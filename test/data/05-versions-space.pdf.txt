Three architectural decisions added. The core product, RAG architecture, LLM routing, and roadmap from 
v4.1 are unchanged. 
 
NEW IN v4.2 
Section 6 -- AI Quality Control Framework 
Six-layer quality control pipeline added. Layer 1-3 are preventive (RAG, structured prompts, memory 
context). Layer 4-5 are reactive feedback loops (student ratings, teacher review queue). Layer 6 is an 
automated evaluation agent (Phase 2) that scores every response before showing it. Quality metrics 
defined. Fine-tuning quality data pipeline added for Phase 3.

Phase 1: worksheet validation loop. JEE/NEET complex solver agent scoped to Phase 3.

4. RAG Architecture -- NCERT-Grounded Answers 
(unchanged from v4.1) 
Every AI answer grounded in uploaded NCERT content -- not LLM training memory. Student asks doubt -> 
question embedded -> NCERT chunks retrieved from pgvector -> injected into prompt. Syllabus updates: re-
upload one PDF. 
 
Step What happens 
1 -- Student 
asks 
Types question or uploads photo of textbook page 
2 -- Check 
cache 
Embed question -> search response_cache for 92%+ similarity match -> return cached if found 
(no LLM call) 
3 -- Retrieve 
NCERT 
Embed question -> cosine search ncert_chunks table -> top 4 passages by subject + class 
filter 
4 -- Inject 
context 
System prompt + student memory + NCERT passages + question. Instruction: Answer using 
ONLY this NCERT content. 
5 -- Stream 
response 
LLM streams response word by word. Cached on completion. XP awarded. Doubt saved to 
DB. 
6 -- Quality 
signal 
Student sees thumbs up/down. Flag button. Rating feeds helpful_rate metric.

($0.10). Syllabus update: re-upload changed PDF, old chunks replaced automatically.

7. Agent Architecture (new in v4.2) 
An agent is an LLM that can plan a sequence of actions, call tools, inspect results, and iterate until a task is 
complete. More powerful than a single LLM call. Also more expensive, slower, and more complex to debug. 
Use agents only where the extra loops justify the cost.

autonomous agents in Phase 3.

7C. Worksheet validation agent -- Phase 1 implementation 
The one agent pattern worth adding in Phase 1. A teacher who receives one factually incorrect question 
loses trust permanently. The validation step costs approximately Rs.0.08 per worksheet and prevents this. 
 
// server/routes/ai.ts -- worksheet route (updated) 
 
app.post('/api/ai/worksheet', async (c) => { 
  const body = await c.req.json() 
  const prompt = getWorksheetPrompt(body) 
 
  // Step 1: Generate worksheet questions 
  const raw = await llmComplete(MODEL.worksheet, [{role:'user',content:prompt}], 4096) 
  const worksheet = JSON.parse(raw) 
 
  // Step 2: Validate each question (self-check agent loop) 
  const validated = [] 
  for (const section of worksheet.sections) { 
    const checkedQuestions = [] 
    for (const q of section.questions) { 
      const checkPrompt = `Is this question factually correct and NCERT-aligned for 
        Class ${body.className} ${body.subject}? 
        Question: ${q.question} 
        Answer: ${q.answer} 
        Reply with ONLY: VALID or INVALID: <reason>` 
      const check = await llmComplete(MODEL.validation, 
[{role:'user',content:checkPrompt}], 100) 
      if (check.startsWith('VALID')) { 
        checkedQuestions.push(q) 
      } else { 
        // Regenerate this question only 
        const regenPrompt = `Generate one replacement question. ${check} -- fix this 
issue.` 
        const replacement = await llmComplete(MODEL.worksheet, 
[{role:'user',content:regenPrompt}], 300) 
        checkedQuestions.push(JSON.parse(replacement)) 
      } 
    } 
    validated.push({...section, questions: checkedQuestions})

Page 14   |   April 2026   |   padee.ai 
  } 
 
  return c.json({...worksheet, sections: validated}) 
}) 
 
Validation model: Llama-3.1-8B (Rs.0.004/1K tokens). At 15 questions per worksheet, average 100 tokens 
per validation call = Rs.0.006 per worksheet for validation. Negligible cost, high trust value.

13. Open Questions (updated) 
 
1. Monetisation entry point: fully free to maximise student data for quality improvement, or freemium 
with 10-doubt cap to test willingness to pay early? 
2. Worksheet validation agent false positive rate: at what rejection rate does the regeneration loop 
create teacher-visible delays? Need to test with real worksheets before setting threshold. 
3. Fine-tuning trigger: what is the minimum labelled dataset size before fine-tuning Llama-8B produces 
measurable quality improvement over the base model on CBSE tasks? Likely 5,000-10,000 pairs but 
needs experiment.
